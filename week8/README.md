# Multi  мультиплеєр через інтернет (Week 08, Unreal Engine 5.7)

Це проєкт, де гравці можуть **створити гру (host)** і **приєднатися до чужої гри (join)** через інтернет,
навіть якщо вони на різних мережах. Працює на бекенді **Steam**.

## Який бекенд і чому

Я вибрала **Steam (Online Subsystem Steam)**, бо:

- він **безкоштовний**  для розробки використовується тестовий App ID `480` ( Spacewar),
  тому не треба нічого публікувати чи платити;
- гравці на **різних мережах** з’єднуються через Steam, тому **не треба** відкривати порти на роутері;
- налаштувати простіше, ніж EOS (не треба реєструвати продукт і акаунт розробника Epic).

## Як працює вхід 

Авторизація  через **сам Steam**. У кожного гравця має бути:

1. відкритий Steam-клієнт,
2. вхід у **справжній акаунт** .

Гра бере Steam-ID з клієнта. Ніяких окремих паролів чи ключів у проєкті немає.

## Що вміє гра

- **Host** - створюєш сесію (можна вибрати максимум гравців) і потрапляєш на карту як сервер.
- **Refresh** - шукає відкриті сесії й показує їх списком.
- **Join** - біля кожної сесії є своя кнопка Join тиснеш і тебе переносить на карту хоста.

Меню з’являється само при запуску гри. Усе зроблено кодом C++.



## Відомі обмеження 

- **Тестовий App ID 480** - тільки для розробки, не для релізу. Щоб не потрапляти у чужі випадкові
  лобі Spacewar, сесії позначені ключовим словом `MultiFreeForAll`.
- **Потрібен Steam** - обидва гравці мусять мати відкритий Steam і справжній акаунт.
- **У редакторі Unreal Steam НЕ вмикається** - це відома поведінка. Тестувати треба **тільки в
  спакованому білді** -. У редакторі меню - лише для перевірки вигляду.
- **Немає host migration** - якщо хост вийшов, сесія закривається.
- **Обидва гравці мають запускати ОДНУ й ту саму збірку** - якщо версії різні, приєднання мовчки не спрацює.
- Немає паролів на сесії й фільтра за регіоном; кількість гравців обмежена =.

- Плагіни: `Multi.uproject` (OnlineSubsystem, OnlineSubsystemSteam).
